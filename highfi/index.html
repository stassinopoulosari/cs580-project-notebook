From the user’s phone home screen, the application can be launched with the application button (a). Alternatively, the user may utilize the “get moving” button from the home screen widget (b) to quickly launch the application.

Item a: Application Button

Item b: Home Screen Widget

Once the application has been launched, the user is introduced to the home screen of the application. From there, the user can initiate a new commute or review earned rewards. If the user uses the alternative widget launch, the welcome screen is bypassed and the user is moved to the departure selection screen.

Once the application has been launched and the user has commenced a commute from either the main screen of the application or by way of the widget “get moving” button, the user is then able to select a location for departure.

The default departure location is the user’s current location as determined by map software integration.

One alternative is to select a designated “home” location which the user can set up in the options menu.

Another alternative is to select another location using the map. Within this option, the user can also select and designate favorite departure locations.

Once the user has selected a departure location, they are able to select a method of transportation. To complete the commute, the user may need to select multiple methods of transportation.

Lastly, the user must select their desired location. Similar to departure location, there are multiple options to choose from. This would include home, various favorites or the option to select a location from the map.

From there, the user starts the commute. Once the commute is in progress, the user manually toggles “next checkpoint” once each checkpoint has been completed.

Once all checkpoints have been completed, the user has arrived at their desired destination and receives an award within the application which celebrates their achieved commute.

Testing

User Population

We conducted our tests in class on Tuesday, November 12, 2024. Our test population was our classmates, who do fit some of the user personas we envisioned when conceiving the application.

Because our classmates fit the profile of students and of working professionals, they are closest to our Persona 1 and Persona 2 from our proposal.

For our test session, our application was evaluated by five of our classmates, who served as our usability experts. The experts were presented with the high fidelity prototype and invited to explore it and provide feedback. As defined in our initial proposal, our experts were able to consider the broad range of transportation methods available in the Portland metropolitan area. The users had varying degrees of experience with the particular methods of interest, including public transportation, micro-mobility, walking, biking and other methods considered. We ran as many small tests within our application as we were able to afford, considering the time allotted and available for our experts. Our application does not have several highly distinct groups of users and rather focuses on general use for a wide variety of overlapping user types. Our users are expected to have similar needs with variability across priorities for their commute. Our application enables users to then customize the application to their needs so that they can find fun and rewarding ways to enhance their commute and hopefully consider highly sustainable methods of transport.

Usability Goals

The usability goals for the overall project were broadly defined in the second deliverable. For the purposes of testing the high fidelity prototype with our expert users, we focused specifically on these key goals and benchmarks.

These are the usability goals that were tested:

  • Launching a Trip
    • Widget activation
    • Trip initialization from home application screen
  • Selecting departure and destination points.
    • Departure point selection
      • Default and alternative options
    • Destination point selection
      • Default and alternative options
  • Trip type & method selection
    • Selecting the desired method of transportation
  • Canceling a trip
    • Ability to cancel trip
  • Progressing trip to conclusion
    • Ability to mark of checkpoints
    • Reward upon completion of all checkpoints

Usability Evaluation (Plan & Results)

We took notes and made observations in line with our usability evaluation plan. We also provided users with a survey to complete. We limited the length of the survey to minimize survey fatigue and facilitate thoughtful responses.

  1. Does the design keep the user informed about what they are doing, progress and status?
    1. Does the user navigate without difficulty?
      • Yes, the users were able to navigate through the application without difficulty. The users benefited from the design which made navigation obvious and clear to the users.
    1. Are there areas where additional direction or guidance is needed?
      • There were some areas for improvement however most issues were related to features that are not yet available or functional. The users had to be advised that some features were not yet functional.
  2. Is the terminology familiar and easily understood? Is the terminology consistent throughout the user experience?
    1. The users were able to easily understand the user interface and navigate with limited prompting. The design was successful in creating an intuitive experience for the user which made the learning curve for usage very minimal. We aim to further improve this with guided walkthroughs and potentially tooltips.
  3. Is the user able to address all preferences when setting up a trip?
    1. Start location
      • All users were able to successfully commence a trip.
    2. End location
      • All users were able to successfully navigate through the trip set-up and commence the commute.
    3. Conclude the trip
      • All users were able to successfully conclude their trip through checkpoints and claim their reward.
  4. Have we organically prevented errors for the user?
    1. Do we observe the user getting tripped up or lost while setting up a commute?
      • No users were lost when using the application though more interactivity was desired. Some users expressed concern over features that were not yet functional within the prototype.
  5. Is the user expected to remember a bunch of information?
    1. Users were not observed having to memorize information. No users raised concerns related to recalling information when navigating through the application.
    2. 80% of users strongly agreed that the application was obvious and easy to use with the remaining 20% agreeing.
  6. Is the application flexible and easy to use?
    1. Users reported ease of use and intuitive application navigation.
    2. 40% of users agreed that the application was easy to navigate and use. The remaining 60% strongly agreed.
  7. Rank aesthetic and cognitive load
    1. 20% of users found the aesthetic to be 4 out of 5 on a scale of poor to excellent.
    2. The remaining 80% found the aesthetic to be excellent.
  8. Are errors clear and precisely identify problems?
    1. There were no immediate issues with errors and problem identification. We did observe some concerns about pending features which were not yet functional at the time of testing.
  9. Is extra help and documentation required? Is the application intuitive?
    1. Users were able to navigate without extra guidance.
    2. We intend to provide the users with both a walkthrough video guide and written documentation on how to initiate and use the application.

Results & Analysis

Users felt that the application was obvious in how it is intended to help facilitate and plan a commute. 20% of users agreed and 80% strongly agreed.

Users agreed (20%) and strongly agreed (80%) that the application is a fun way to plan a commute and would inspire them to use more sustainable methods of transportation. This was a very important finding which supported the sustainability goal which inspired this project.

Users agreed (40%) and strongly agreed (60%) that the application was easy to navigate. This was also supported while observing our expert users interacting with the application. Navigation was simple, obvious and direct.

Users felt the rewards system was a compelling and persuasive aspect of the application. This was ranked as the most encouraging aspect of the application with all survey participants considering this one of their top 3 elements. The second and third most valued elements were ease of use and improving commute time.

Lastly, the aesthetic of the application was held in high regard. 20% of users gave the aesthetic a 4 out of 5 rating. The remaining 80% of users gave the aesthetic a perfect score.

Further analysis and assessment of the results is warranted and will be incorporated into the final presentation.
