Usability Goals & Benchmark Tasks

Task 1: Launching a Trip from the Home Screen

  • Overview

    Allow users to quickly start a trip using a home screen widget or directly from the app.

  • Outline

    1. Widget Activation:
      • The user taps on the widget or icon from the home screen.
      • The widget shows quick actions like “Start Trip” and presents a quick summary (destination, trip type, etc.).
    2. Trip Initialization:
      • The app loads the current or default departure and destination options.
      • The user is asked to confirm the trip and transport method or edit these selections.
    3. Feedback:
      • Provide immediate feedback (ETA, transport type, rewards preview) once the trip is confirmed.
      • Start trip navigation or directions based on user preference.
    4. Background Integration:
      • The app continues running the trip in the background, sending periodic updates or notifications if relevant.

Task 2: Selecting Departure and Destination Points

  • Overview

    Users can set their departure location (current location, alternative, or home) and destination from within the app.

  • Outline

    1. Departure Point Selection:
      • Default Option: App automatically selects “current location” via GPS.
      • Alternative Option: User can manually enter or select an alternative starting point.
      • Home Location: Home address saved as the default alternative departure point.
    2. Destination Point Selection:
      • Search or browse feature for selecting destinations (e.g. saved locations, recent destinations, calendar appointment locations).
      • Integration with maps for a visual selection of destination.
    3. Confirmation:
      • The user confirms departure and destination points.
      • The app provides options to adjust departure or arrival times if necessary.
    4. Next Steps:
      • The user moves to the trip method selection after departure/destination is confirmed.

Task 3: Trip Type and Transport Method Selection with Sustainability Ranking

  • Overview

    Users can choose from various transport options with rankings based on sustainability, which also influence reward points.

  • Outline

    1. Transport Methods:
      • Present users with transport options: walking, biking, public transit, carpooling, driving (electric vehicles may rank higher).
    2. Ranked Options:
      • Options ranked based on sustainability (e.g. walking or biking receives highest score, driving gasoline-powered vehicles ranks lower).
      • Visual badges or markers (e.g. green for eco-friendly, red for less eco-friendly) highlight the ranking system.
    3. Rewards System:
      • Clear indication of the rewards or achievements each option offers.
      • An explanation of why certain modes receive higher- or lower-level rewards (e.g. lower emissions, healthier choices).
    4. Selection and Confirmation:
      • User chooses the preferred transport method.
      • The app shows trip details, ETA, and rewards tied to the selection.

Task 4: Canceling a Trip

  • Overview

    Users can cancel trips anytime from the widget or app.

  • Outline

    1. Widget Cancel Option:
      • A “Cancel Trip” button is available on the widget for quick access.
      • Confirmation prompt appears asking the user to confirm the cancellation.
    2. App Cancel Option:
      • The user can access the trip details from within the app.
      • Select the “Cancel Trip” button and confirm the action.
    3. Post-Cancellation Actions:
      • Provide a short feedback form (optional) to understand cancellation reasons.
      • If rewards were allocated based on trip progress, notify the user if points are forfeited or saved.

Task 5: Changing a Trip Mid-Way

  • Overview

    Users can modify an ongoing trip from the widget or app, changing destinations or transport methods.

  • Outline

    1. Accessing Trip Modifications:
      • From the widget or the app, the user accesses the ongoing trip details.
    2. Options for Changes:
      • Change Destination: Allows users to input a new destination.
      • Change Transport Method: Users can switch from one transport method to another (e.g. from public transit to bike or from walking to public transit).
      • Terminate Trip: Users may fully cancel the trip, similar to the cancellation process.
    3. Recalculation:
      • Once a change is made, the app recalculates the route and provides new details like ETA and rewards changes.
    4. Confirmation:
      • The app asks for final confirmation before applying the change.
      • Notifications update the user with new trip details.

Task 6: Viewing and Managing the Reward System

  • Overview

    Users can review their transportation-related accomplishments through a rewards system in the app.

  • Outline

    1. Accessing the Rewards Section:
      • From the app’s main menu, the user selects the “Rewards” or “Achievements” section.
    2. Rewards Overview:
      • Display current medal cabinet, past trips, and sustainability accomplishments (e.g. emissions saved, calories burned).
    3. Detailed Breakdown:
      • Users can click on specific trips to see how many medals/gems/rewards were earned and why (e.g. eco-friendly choice, distance covered).
